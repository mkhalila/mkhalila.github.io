Initiative & Creative Problem Solving

We were working in a team of four to create a Shark Tracker application based on real-time data provided by OCEARCH. One requirement of the software was to determine if a Shark was on land or at sea given it's latitude and longitude. A limitation was that we were prohibited from using paid APIs, we also - at that point - had no experience of integrating APIs; and a further restriction was to implement this feature with minimum overheads without sacrificing accuracy.

Therefore I came up with a solution that would retrieve a 1*1 pixel image from the Google static maps API and pre-configured all land areas to have a blue RGB value of 0 and all water to have a value of 255. The algorithm would then obtain the pixel and determine - based on the RGB value - whether the shark was at sea or not.
